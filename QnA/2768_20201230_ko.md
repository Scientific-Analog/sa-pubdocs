# Q. MODELZEN이 불필요하게 소자특성 시뮬레이션을 다시 수행하는 것을 방지하는 방법

MODELZEN 실행을 여러번 하면, MODELZEN은 이전에 수행했던 소자특성 시뮬레이션 결과들을 재활용해서 모델을 빨리 생성하는 것을 보았습니다. 하지만, technology configuration 파일에 어떤 수정이라도 가하면, 그 수정 내용이 이전 소자특성 시뮬레이션 결과들에 전혀 영향을 주지 않는 아주 사소한 것이라 하더라도, MODELZEN은 보수적으로 이전 소자특성 시뮬레이션 결과들이 더 이상 유효하지 않다고 가정하고 모든 소자특성 시뮬레이션을 다시 수행하는 것 같습니다. 이것은 물론 올바른 결과를 보장하는 안전한 방법이라는 데에는 동의하지만, 가끔은 시간을 불필요하게 낭비하는 요소가 되는 것 같습니다.
사용자가 직접 MODELZEN에게 technology configuration 파일이 수정되었지만, 그 수정 내용이 이전 소자특성 시뮬레이션 결과에 영향을 미치지 않으니 그것들을 재활용해도 괜찮다고 알려주는 방법이 있을까요?

## Answers

### A1

상황에 따라서 두가지 방법이 가능합니다. 한가지는 임시적인 방법이고, 다른 한가지는 영구적인 방법입니다.
만일 technology configuration 파일에 간단한 수정을 하셨고, 그 수정 때문에 소자특성 시뮬레이션을 다시 수행하는 것을 임시적으로 피하고 싶으시다면, technology configuration 파일의 수정시각을 과거로 돌려서 MODELZEN을 속이는 방법이 있습니다. 이것은 MODELZEN이 소자특성 시뮬레이션을 수행해야 하는지 여부를 파일들의 수정시각을 비교해서 정한다는 것을 이용한 방법입니다. 예를 들어, 마지막 MODELZEN 소자특성 시뮬레이션을 2020년 12월 28일 어느 시각에 수행했었다면, 아래와 같이 리눅스의 `touch` 명령을 사용해서 technology configuration 파일(예를 들면, `tech_config.py` 파일)의 수정시각을 의도적으로 같은 날의 0시 0분 0초로 설정할 수 있습니다.

```
touch -d "2020-12-28" tech_config.py
```

다른 한편, technology configuration 파일의 내용 중 공정 정보를 정의하고 있는 `dev_model`, `devo_devicemap`, `devo_subcktmap` 등의 내용이 충분한 과정을 통해 검증되었고, 이제는 더이상 수정이 필요없을 것이라는 확신이 드는 단계에 도달하면, 지금까지 얻은 소자특성 시뮬레이션 결과들을 시뮬레이션 금고(*simulation vault*)로 옮겨 영구적으로 신뢰할 수 있는 결과로 취급하게 할 수 있습니다. MODELZEN은 시뮬레이션 금고에 담긴 소자특성 시뮬레이션 결과들은 technology configuration 파일의 수정시각에 관계없이 항상 신뢰하며, 따라서 항상 재활용합니다.
기존의 소자특성 시뮬레이션 결과들을 시뮬레이션 금고로 옮기려면, MODELZEN 시뮬레이션 디렉토리를 새 위치로 복사하고, 그 새 위치경로를 technology configuration 파일에 정의하면 됩니다. MODELZEN 시뮬레이션 디렉토리는 MODELZEN이 수행한 소자특성 시뮬레이션 결과들을 담고 있는 디렉토리로서, 그 경로의 기본값은 MODELZEN을 GLISTER 상에서 실행했다면 `./modelzen.run/sim`, MODELZEN을 리눅스 커맨드라인에서 실행했다면 `./modelzen.sim`입니다. 예를 들어, `./modelzen.run/sim`에 위치한 MODELZEN 시뮬레이션 디렉토리를 새로운 위치인 `/PATH/sim.vault`로 복사하려면, 아래처럼 리눅스의 `cp` 명령을 `-R` 옵션과 함께 사용하시면 됩니다.

```
cp -R ./modelzen.run/sim /PATH/sim.vault
```

그 다음에는 이 시뮬레이션 금고의 경로 정보를 설정하기 위해, technology configuration 파일에 아래 라인을 삽입하시면 됩니다.

```
devo_options['sim_vault'] = '/PATH/sim.vault'
```

이 시뮬레이션 금고 기능은 같은 공정 셋업을 사용하는 복수의 사용자 또는 팀 간에 소자특성 시뮬레이션 결과들을 공유하고 재활용할 수 있는 효과적인 방법이 될 수 있습니다. 하지만, 이 시뮬레이션 금고에 담긴 소자특성 시뮬레이션 결과들을 맹목적으로 신뢰함으로써 발생할 수 있는 다양한 문제를 방지하기 위해서는 매우 세심한 주의가 필요합니다. 즉, 이 시뮬레이션 금고를 사용하는 사용자들은 그 안에 담긴 소자특성 시뮬레이션 결과들이 최신 정보로 업데이트된 유효한 결과라는 사실을 책임의식을 갖고 보장할 수 있어야 합니다.
