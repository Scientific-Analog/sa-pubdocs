# Q. 'replay_all' primitive를 사용할때 지원하지 않는 문자를 사용한 신호 이름을 정상적인 이름으로 바꾸는 방법

테스트벤치 스키메틱에서 `'replay_all'` primitive를 사용해 이전에 시뮬레이션한 FSDB 형식의 파형들을 재생하고 있습니다. 문제는, 몇몇 파형들의 이름이 Verilog에서 지원하지 않는 문자를 사용하고 있습니다. 예를 들면, `"v1:p"`처럼 콜론(`:`)을 포함한 이름입니다. 이때, `'replay_all'` primitive의 포트에 연결된 net들도 같은 이름을 가지게 되면서 문제가 발생합니다. 지원하지 않는 문자를 사용한 신호 이름들을 정상적인 이름들로 자동변환하는 방법이 있을까요?

## Answers

### A1

예, 방법이 있습니다. XMODEL 2021.09 Release부터는 `'replay_all'` primitive의 터미널 이름들을 원하는 다른 이름으로 변환할 수 있는 사용자 정의 함수인 `xmodelMapTermNameReplayAll()` 함수를 지원합니다.
이 함수의 기본정의는 아래 코드를 사용하여 신호 이름에 담긴 모든 콜론 문자 `':'`를 `'__'`로 바꾸는 것입니다.

```
; xmodelMapTermNameReplayAll: user-customizable function for mapping terminal names for replay_all primitive
procedure( xmodelMapTermNameReplayAll( signame )
    signame = pcreReplace( pcreCompile( ":" ) signame "__" 0 )
    signame
)
```

사용자는 Cadence Virtuoso의 초기화 파일인 `.cdsinit` 파일 내에서 XMODEL/GLISTER의 context 파일인 `xmodel.cxt`가 로딩된 이후 위치에서 이 함수를 재정의할 수 있습니다. 예를 들면, 콜론 문자 뿐만 아니라 모든 `'&'` 문자들도 `'__'`로 바꾸고 싶다면, 아래와 같이 `xmodelMapTermNameReplayAll()` 함수를 재정의할 수 있습니다.

```
; xmodelMapTermNameReplayAll: mapping all ':' and '&' characters to '__'
procedure( xmodelMapTermNameReplayAll( signame )
    signame = pcreReplace( pcreCompile( ":" ) signame "__" 0 )
    signame = pcreReplace( pcreCompile( "&" ) signame "__" 0 )
    signame
)
```

위의 코드들에 사용된 `pcreReplace()`와 `pcreCompile()` 함수를 포함한 Cadence SKILL 언어에 대한 자세한 설명은 관련 Cadence 문서들을 참고하시기 바랍니다 (예를 들면, Cadence SKILL Language User Guide 또는 Cadence SKILL Language Reference).
