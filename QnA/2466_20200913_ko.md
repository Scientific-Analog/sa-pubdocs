# Q. Elaboration 단계에서 발생하는 포트 사이즈 미스매치 에러

시뮬레이션을 실행시켰을때 elaboration 단계에서 아래와 같은 에러/경고 메시지가 납니다.

```
ncelab: *E,CUVMPW: port sizes differ in port connection (1/137).
```

무슨 뜻이고, 어떻게 고쳐야 하나요?

## Answers

### A1

일반적으로 이 메시지는 어느 포트에 다른 비트 길이를 가진 신호가 연결되어 있다는 뜻입니다. 하지만, 대상 포트가 `xreal` 또는 `xbit` 타입의 포트라면, 다른 타입의 신호를 연결하셨을 가능성도 함께 고려하셔야 합니다. 다른 타입의 신호가 다른 비트 길이의 신호로 인식되는 경우도 있으니까요. 예를 들면, `xreal` 타입 포트에 `wire/reg` 타입 신호를 연결하셨거나, 그 반대의 경우일수도 있습니다. 서로 연결된 신호와 포트 쌍이 같은 타입을 갖고 있는지 세심히 확인해보세요.
말씀하신 메시지는 NC-Verilog 또는 XCelium이 리포트하는 메시지입니다. 참고로, 같은 문제에 대해 VCS는:

```
Warning-[PCWM-W] Port connection width mismatch
```

메시지를 리포트하고, Questa 또는 ModelSim은:

```
** Warning: (vsim-3015): [PCDPC] - Port size (137) does not match connection size (1) for port 'a'.
```

메시지를 리포트합니다.
사용하시는 SystemVerilog 시뮬레이터에 따라서, 그리고 함께 사용하는 옵션에 따라서, 시뮬레이터는 이 문제를 경고 또는 에러로 리포트합니다. 이러한 연결 문제가 있으면 XMODEL 시뮬레이션을 제대로 동작하지 않고, 또 가끔은 SIGSEGV 등 직접적인 원인을 알기 힘든 에러 메시지를 내면서 실행을 멈추기도 하기 때문에, 이 메시지의 중요도를 경고에서 에러로 상향하시는 것을 추천드립니다. 그 방법은 SystemVerilog 시뮬레이터에 따라 다르기 때문에 자세한 정보는 각 시뮬레이터의 매뉴얼 문서를 참고하시기 바랍니다.
