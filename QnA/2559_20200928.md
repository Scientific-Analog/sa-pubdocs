# Q. Moving the location of the XMODEL installation directory

In my company, the CAD tools are first installed to a local directory and then moved to the central place. But I found that it does not work for XMODEL as some of its files bear the absolute path name of the initial installation directory. Is there a way to reconfigure the XMODEL installation path to a new one?

## Answers

### A1

As you noticed, XMODEL hardwires the absolute path information of its location into some of its files during the installation, and XMODEL would no longer work when you move its installation directory to a different location. Unfortunately, there is currently no way to reconfigure this information once the installation is done.

The easiest remedy is to install XMODEL directly at the `<final path>` using the following command:

```
$ xmodel_2020.09.x86_64.sh -b -p <final_path>
```

where the actual name of the installation script may differ depending on the XMODEL release.

If your situation does not permit this, an alternate remedy is to install XMODEL at an `<initial path>` assuming that it will be moved to a different `<final path>` later using the `'-x'` option:

```
$ xmodel_2020.09.x86_64.sh -b -p <initial_path> -x <final_path>
```

Then, you can move the XMODEL installation directory from `<initial_path>` to `<final_path>` as below:

```
$ mv <initial_path> <final_path>
```
