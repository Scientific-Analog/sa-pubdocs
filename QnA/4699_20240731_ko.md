# Q. 파형 정보를 CSV 포맷으로 export하는 법

XMODEL로 시뮬레이션한 파형을 다른 툴로 옮겨서 후처리를 하고 싶습니다. 파형 정보를 CSV와 같은 텍스트 형식으로 export하는 방법이 있을까요?

## Answers

### A1

예, XMULAN 파이썬 라이브러리를 활용하면 JEZ 형식의 파형 파일을 읽어서 그 안에 담겨있는 정보를 또다른 형식으로 export하는 것이 가능합니다. 아래의 파이썬 스크립트는 `'xmodel.jez'`이라는 이름의 파형 파일에 담겨있는 `'TOP.A'`라는 이름의 신호의 파형 정보를 CSV 형식으로 export하는 예를 보여줍니다.

**export\_waveform.py:**

```
#!/usr/bin/env xmodelpy

import xmulan
import csv

waveform_filename = "xmodel.jez"
signal_name       = "TOP.A"             
output_filename   = "data.csv"

# read waveform data
row = xmulan.rowml().readmeas(waveform_filename)
trace = row[signal_name]
wf = trace.get_waveform()
T, V = wf.sweep[0,:], wf

# write time-value pairs in CSV format
csvfile = open(output_filename, "w")
writer = csv.writer(csvfile)
writer.writerows(zip(T, V))
csvfile.close()
```

이 스크립트는 `xreal`, `xbit`, `real`, `wire`, `reg` 및 `bit` 타입의 신호에 똑같이 사용될 수 있습니다. 하지만, `xreal` 타입의 신호에 대해서는 그 파형의 시간-값 정보를 추출하는 복수의 방법이 가능합니다.
예를 들어, 위 스크립트에 보인 것과 같이 `get_waveform()` 함수를 아무 인자없이 사용하면, 그 함수는 원래의 신호 파형을 piecewise-linear 방식으로 표현했을 때 잘 근사할 수 있는 점들의 시간-값 정보를 리턴합니다. 이때, 리턴되는 점들은 일정하지 않은 시간 간격을 가질 수 있습니다.

```
wf = trace.get_waveform()
T, V = wf.sweep[0,:], wf
```

하지만, 어떤 경우에는 일정한 시간 간격의 점들의 시간-값 정보를 추출하고 싶을 때가 있습니다. 그 경우에는 아래처럼 `mode="fixedstep"` 인자를 사용하고, 원하는 시간 간격값을 `'tstep'` 인자를 통해 정해주면 됩니다.

```
wf = trace.get_waveform(mode="fixedstep", tstep=1e-12)
T, V = wf.sweep[0,:], wf
```

또한, 임의의 시간 지점에서 샘플한 파형의 값을 추출할 수도 있습니다. 아래는 0부터 100ns 범위 내에서 20ps 간격으로 샘플링한 신호 파형의 값을 얻는 방법을 보여줍니다.

```
import numpy as np

T = np.r_[0.0:100e-9:20e-12]
V = trace.eval_at(T)
```
