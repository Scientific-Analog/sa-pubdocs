# Q. XMODEL 설치 이후 설치경로 변경하는 방법

우리 회사에서는 CAD 툴을 먼저 임의의 위치에 먼저 설치한 후, 이를 추후 공유폴더로 옮기는 절차를 밟습니다. 하지만, XMODEL의 경우, 설치과정에서 초기 설치폴더의 절대경로 정보가 내부파일들에 기록되어, 설치된 XMODEL 패키지를 추후 다른 경로로 옮기면 더이상 정상동작하지 않네요. 이 설치경로 정보를 재정의하는 방법이 있나요?

## Answers

### A1

말씀하셨듯이 XMODEL은 설치 과정 중 설치 폴더의 절대 경로 정보를 내부 파일들에 기록하기 때문에 설치 폴더를 추후에 다른 위치로 바꾸면 정상적으로 동작하지 않습니다. 그리고, 아직 설치 후에 이 설치 경로 정보를 재설정할 수 있는 방법은 없습니다.

이 문제를 해결하는 가장 쉬운 방법은 아래 명령을 사용해 최종 설치 경로인 `<final path>`에 직접 XMODEL을 설치하는 것입니다.

```
$ xmodel_2020.09.x86_64.sh -b -p <final_path>
```

참고로, 위의 설치 스크립트 이름은 XMODEL 버전에 따라 변경될 수 있습니다.

만약 최종 경로에 설치를 바로 할 수 없는 상황이라면, 아래처럼 `'-x'` 옵션을 사용해 최종 설치 경로가 `<final path>`로 바뀔 것을 가정해 임시 설치 경로인 `<initial path>`에 설치를 진행할 수 있습니다.

```
$ xmodel_2020.09.x86_64.sh -b -p <initial_path> -x <final_path>
```

이런 식으로 설치가 된 후에는, XMODEL 설치 폴더의 위치를 `<initial_path>`에서 `<final_path>`로 변경하실 수 있습니다.

```
$ mv <initial_path> <final_path>
```
