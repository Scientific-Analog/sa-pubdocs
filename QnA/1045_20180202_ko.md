# Q. XMODEL 새 버전으로 바꾸었을때 발생하는 시뮬레이션 에러

XMODEL 2017.06 버전에서는 잘 도는 시뮬레이션이, 2018.01 버전으로 바꾸면 아래의 메시지를 내며 멈춥니다.

```
*** ERROR: error while parsing the configuration file:
    /data1/Tools/XMODEL/2_xmodel_2018.01_x86_64/etc/default_config.py

In line #19:
    from devo_util import regex, wildcard

ImportError: No module named devo_util
```

## Answers

### A1

해당 문제는 2017.06 버전의 환경세팅과 2018.01 버전의 환경세팅을 혼용해서 발생한 것으로 생각됩니다. 예를 들어, 언급하신 에러메시지는 2018.01 환경세팅을 한 후에 2017.06 실행파일을 실행하면 발생할 수 있습니다.  
한가지 가능성은, 버전을 2018.01으로 바꾸실때, `$XMODEL_HOME` 환경변수만 2018.01 버전의 것으로 바꾸시고, `$PATH` 환경변수는 안 바꾸셔서, XMODEL 시뮬레이션을 할때 여전히 2017.06 버전의 실행파일을 사용하고 계실 가능성이 있습니다.  
확인하는 방법은, 커맨드 라인에서:

```
echo $XMODEL_HOME
```

와

```
which xmodel
```

를 실행하셔서 둘 사이에 XMODEL 버전의 차이가 있는지 보세요.
